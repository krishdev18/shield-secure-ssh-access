# 🛡️ S.H.I.E.L.D Secure Access Upgrade (Telnet → SSH)

## 📌 Overview
This project simulates a secure remote access upgrade in a S.H.I.E.L.D network.  
Initially, Telnet was used for device management, which is insecure.  
The network was upgraded to SSH to ensure encrypted communication.

---

## 🌍 Scenario
S.H.I.E.L.D manages critical infrastructure across:
- Avengers Tower
- Stark Lab
- Helicarrier Control

Due to security threats from Hydra, Telnet access was replaced with SSH.

---

## 🧠 Objectives
- Configure Telnet (insecure)
- Upgrade to SSH (secure)
- Enable remote login authentication
- Understand encryption vs plaintext communication

---

## 🏗️ Topology
- 1 Router (SHIELD Core)
- 1 Switch
- Multiple PCs (Admins)

---

## 🔐 Technologies Used
- Telnet
- SSH
- Cisco CLI
- Packet Tracer

---

## 🚀 Outcome
- Secure remote access implemented
- Encrypted communication enabled
- Unauthorized access prevented

---

## 🧪 Testing
- Telnet connection tested (insecure)
- SSH connection tested (secure)

---

## 📂 Project Structure
configs/ → device configurations  
docs/ → explanation and troubleshooting  
tests/ → validation  
topology/ → packet tracer file  

---

## 💡 Key Learning
Telnet sends data in plaintext, while SSH encrypts communication, making it secure for real-world networks.
